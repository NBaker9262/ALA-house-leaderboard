## Firebase Setup

Firebase is the only backend. There are no Cloud Functions or custom servers.

Required Firebase services:

- Authentication with Email/Password enabled.
- Firestore Database enabled.

Confirm CLI login:

```powershell
firebase login:list
firebase use
```

Expected project:

```text
ala-house-leaderboard
```

Deploy rules/indexes after changing `firestore.rules` or `firestore.indexes.json`:

```powershell
npm run firebase:deploy
```

## Firestore Data Model

Main score document:

```text
leaderboard/scores
```

Important subcollections:

```text
leaderboard/scores/auditLog
leaderboard/scores/pendingProposals
```

Other collections:

```text
userProfiles
studentDirectory
eventTags
eventTagProposals
contactMessages
systemBackups
```

The public leaderboard should only need the `leaderboard/scores` document.

## Roles

Role documents live at:

```text
userProfiles/{firebaseAuthUid}
```

Basic fields:

```text
uid: Firebase Auth UID
email: user email
name: display name
role: superadmin | admin | member | viewer
active: true
permissions: map of permission booleans
```

Role meanings:

- `superadmin`: everything, including users, backups, reset, imports, exports.
- `admin`: scoring, approvals, imports, exports, history.
- `member`: scoring and student lookup.
- `viewer`: optional suggestion-only role. Viewers can suggest points but cannot directly edit scores.

## User Creation

User creation stays inside the app.

1. Open the full iPhone/control app.
2. Sign in as a `superadmin`.
3. Open `Admin Management`.
4. Use `Account Management`.
5. Enter name, email, temporary password, and role.
6. Click `Create / Update User`.

This creates the Firebase Auth user and writes/updates `userProfiles/{uid}`.

Password help uses the same superadmin flow. Signed-out users click `Forgot Password?`, which creates a superadmin-visible request in `contactMessages`. In `Admin Management`, superadmins can load requests, send Firebase reset links, generate strong temporary passwords for new users, copy them to the clipboard, and mark requests done. Existing Firebase Auth passwords cannot be directly changed from the browser client; use a reset link for existing accounts.

## Google Sheets Setup

The app uses CSV import/export so it can work without a backend or Google Sheets API OAuth.

### Sheet 1: Student Roster

Create a Google Sheet named:

```text
House Points - Student Roster
```

Headers:

```csv
name,house,studentId,grade,email,active
```

You can start from:

```text
templates/student-roster-template.csv
```

Allowed house values:

- `red`, `panda`, `red panda`
- `white`, `polar`
- `blue`, `grizzly`
- `silver`, `kodiak`

Example:

```csv
name,house,studentId,grade,email,active
Alex Rivera,red,10001,9,alex@example.org,true
Jordan Lee,polar,10002,10,jordan@example.org,true
```

Import roster:

1. In Google Sheets, choose `File -> Download -> Comma Separated Values (.csv)`.
2. In the full control app, open `Admin Management -> Data Import/Export`.
3. Click `Choose Student Roster CSV`.
4. Preview the rows.
5. Click `Import Student Roster`.

The app writes students to:

```text
studentDirectory
```

It also creates `searchPrefixes` for fast lookup by name, email, or student ID.

### Sheet 2: Points Import/Export

Create a Google Sheet named:

```text
House Points - Point Imports
```

Headers:

```csv
house,points,reason,date
```

You can start from:

```text
templates/points-import-template.csv
```

Example:

```csv
house,points,reason,date
red,50,Assembly win,4/29/2026
white,30,Second place,4/29/2026
blue,15,Third place,4/29/2026
silver,10,Fourth place,4/29/2026
```

Import points:

1. In Google Sheets, choose `File -> Download -> Comma Separated Values (.csv)`.
2. In the full control app, open `Admin Management -> Data Import/Export`.
3. Click `Choose Points CSV`.
4. Preview the rows.
5. Click `Import Points`.

Important: point import replaces current totals with totals calculated from the CSV and writes audit rows under:

```text
leaderboard/scores/auditLog
```

### Optional: Sheet 2 Pulls Roster From Sheet 1

If you want Sheet 2 to be owned by you and pull roster data from Sheet 1:

1. Open Sheet 2.
2. In cell `A1`, use:

```text
=IMPORTRANGE("PASTE_STUDENT_ROSTER_SHEET_URL_HERE","Sheet1!A:F")
```

3. Allow access when Google Sheets asks.
4. Keep a separate tab in Sheet 2 named `Point Imports` with:

```csv
house,points,reason,date
```

## Export Back To Google Sheets

In the full control app:

1. Open `Admin Management -> Data Import/Export`.
2. Use one of:
   - `Export Points CSV`
   - `Export Student Roster CSV`
3. In Google Sheets, choose `File -> Import`.
4. Upload the CSV.
5. Choose `Replace spreadsheet` or `Insert new sheet`.

## iPhone Build And Sideload

The iPhone project is already scaffolded with Capacitor iOS.

Every time you change `iPhone Sideload App/control1.html`, run from repo root:

```powershell
npm run iphone:sync
```

That does:

1. Copies `control1.html` to `www/index.html`.
2. Syncs web assets into `ios/App/App/public`.

### Copy To Mac

Copy the whole `iPhone Sideload App` folder to the Mac, including:

- `ios/`
- `www/`
- `control1.html`
- `package.json`
- `package-lock.json`
- `capacitor.config.json`

### Build In Xcode

On the Mac:

```bash
npm install
npm run build:ios-ready
npm run open:ios
```

In Xcode:

1. Select the `App` target.
2. Open `Signing & Capabilities`.
3. Choose your Apple ID Personal Team.
4. Keep bundle identifier unique, currently `com.noahm.housepoints`.
5. Connect the iPhone by cable.
6. Enable Developer Mode on iPhone if prompted.
7. Select the real iPhone device.
8. Press Run.

### Export IPA

1. In Xcode device selector, choose `Generic iOS Device` or connected iPhone.
2. Choose `Product -> Archive`.
3. Open `Window -> Organizer -> Archives`.
4. Select latest archive.
5. Choose `Distribute App`.
6. Export a development-signed `.ipa`.

### Install With AltStore

1. On Windows, keep AltServer running as Administrator.
2. Install Apple-site iTunes and iCloud, not Microsoft Store versions.
3. Keep iPhone on the same Wi-Fi as AltServer or connected by USB.
4. Open AltStore on iPhone.
5. Go to `My Apps`.
6. Add/sideload the `.ipa`.
7. Trust the profile on iPhone if prompted.

Free Apple ID reminder: refresh apps in AltStore before 7 days.

1. Open `leaderboard.html` on the display/projector.
2. Open the iPhone/full control app.
3. Sign in.

## Troubleshooting

If app changes do not show on iPhone:

```powershell
npm run iphone:sync
```

Then reopen Xcode, clean build folder, and rerun.

If Firebase access fails:

```powershell
firebase login:list
firebase use ala-house-leaderboard
npm run firebase:deploy
```

If a user cannot sign in:

1. Confirm they exist in Firebase Authentication.
2. Confirm `userProfiles/{uid}` exists.
3. Confirm `role` is one of `superadmin`, `admin`, `member`, `viewer`.
4. Confirm `active` is `true`.

If student lookup fails:

1. Re-import the student roster CSV.
2. Confirm rows exist in `studentDirectory`.
3. Confirm each row has `searchPrefixes`.
4. Confirm the signed-in role has `studentLookup`.

If Google Sheets import looks wrong:

1. Confirm the header row exactly matches the expected columns.
2. Download as CSV, not XLSX.
3. Check house names are one of the allowed values.
4. For points import, remember the CSV totals replace current totals.
